Build app

→ ShadCN: <Card> + <Collapsible>

The build checklist as a timeline: a head node anchors the top, the rail fills and darkens as each step moves queued → in progress → done (one green, #115B3B, at opacity steps), and a terminal “App built” node resolves to a check when every step finishes. Completed rows keep their primary text. Two state axes: collapse (true | false) and state (resting | hover). Replaces the legacy checkbox-based plan.

Thinking

→ ShadCN: small inline indicator (custom)

Shown while the assistant is working. The Assembly chevron logo builds layer by layer — bottom chevron eases in first, then the middle, then the top — then all three hold briefly and the whole mark resets. The eased transitions between layers keep the build feeling considered (not jittery) while still reading as discrete steps. The motion mirrors the brand mark's own visual language: the three chevrons are layers, so revealing them in sequence reads as on-brand assembly.

Plain prose from the assistant. Sits inline in the chat scroll — no avatar, no bubble — so the assistant's voice reads as the dominant narration. Supports paragraphs, bullets, and bold. For collapsable thought-process content, wrap it in Streaming logic.

Streaming logic

→ ShadCN: <Collapsible> + custom timeline

Instead of streaming a wall of agent thoughts, we bundle the work into a collapsable summary — most users never need to expand it. When opened, it reveals a clean vertical timeline of the steps actually taken: bullets of decisions, references the agent pulled in, and the build checklist it ran against. The Streaming row composes the same primitives the rest of the chat uses, so it stays inline with the conversation.
